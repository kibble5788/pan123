# 获取上传域名接口说明
## 一、接口基础信息
- **API 地址**：GET 域名 + /upload/v2/file/domain
- **核心用途**：用于单步上传流程，获取后续文件上传所需的域名列表（多个域名可任选其一使用）。

## 二、请求参数
### （一）Header 参数
| 名称 | 类型 | 是否必填 | 说明 |
| ---- | ---- | ---- | ---- |
| Authorization | string | 是 | 鉴权 access_token |
| Platform | string | 是 | 固定值：open_platform |

### （二）Body 参数
无

## 三、返回数据
| 名称 | 类型 | 是否必填 | 说明 |
| ---- | ---- | ---- | ---- |
| data | array | 是 | 上传域名列表，支持任选其一用于后续上传操作 |

## 四、请求示例（Curl）
```curl
curl --request GET \
--url https://open-api.123pan.com/upload/v2/file/domain \
--header 'Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5c...(过长省略)' \
--header 'Platform: open_platform'
```

## 五、响应示例
```json
{
"code": 0,
"message": "ok",
"data": [
"https://openapi-upload.123pan.com"
],
"x-traceID": ""
}
```

## 补充说明
- 其他语言请求示例（Java - OkHttp、JavaScript - jQuery、NodeJs - Axios、Python - http.client）可参考官方完整文档。
- 响应中 `code: 0` 表示接口请求成功，`data` 数组返回可用上传域名，需将该域名用于单步上传的后续发起上传操作。
