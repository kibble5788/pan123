# 上传分片接口说明
## 一、接口基础信息
- **API 地址**：POST 上传域名 + /upload/v2/file/slice
- **核心说明**：
  1. 上传域名来源：需使用「创建文件」接口响应中返回的 `servers` 列表中的域名。
  2. 请求格式要求：`Content-Type` 必须为 `multipart/form-data`。

## 二、请求参数
### （一）Header 参数
| 名称 | 类型 | 是否必填 | 说明 |
| ---- | ---- | ---- | ---- |
| Authorization | string | 是 | 鉴权 access_token |
| Platform | string | 是 | 固定值：open_platform |

### （二）Body 参数（form-data 格式）
| 名称 | 类型 | 是否必填 | 说明 |
| ---- | ---- | ---- | ---- |
| preuploadID | string | 是 | 预上传 ID（来源于「创建文件」接口响应） |
| sliceNo | number | 是 | 分片序号，从 1 开始自增（按文件分片顺序依次填写） |
| sliceMD5 | string | 是 | 当前分片的 MD5 校验值 |
| slice | file | 是 | 分片二进制流（需上传对应分片的文件数据） |

## 三、返回数据
- 无额外业务数据，仅返回接口状态信息，响应示例如下：
```json
{
"code": 0,
"message": "ok",
"data": null,
"x-traceID": ""
}
```

## 四、请求示例（Curl）
```curl
curl --request POST \
--url http://openapi-upload-dev.123242.com/upload/v2/file/slice \
--header 'Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5c...(过长省略)' \
--header 'Platform: open_platform' \
--header 'content-type: multipart/form-data' \
--form preuploadID= WvjyUgonimrlI4sjB3sLG5sRBn3x43...(过长省略) \
--form sliceNo= 1 \
--form sliceMD5=58f06dd588d8ffb3beb46ada6309436b \
--form 'slice=@E:\新建文件夹\测试分片.txt.part1'
```

## 补充说明
- 其他语言请求示例（Java - OkHttp、JavaScript - jQuery、NodeJs - Axios、Python - http.client）可参考官方完整文档。
- 接口返回 `code: 0` 且 `message: "ok"` 表示当前分片上传成功，需按序号依次上传所有分片。
