# 删除文件至回收站接口说明
## 一、接口基础信息
- **API 地址**：POST 域名 + /api/v1/file/trash
- **核心说明**：调用该接口删除的文件不会直接永久删除，而是放入云盘回收站中，支持后续恢复操作。

## 二、请求参数
### （一）Header 参数
| 名称 | 类型 | 是否必填 | 说明 |
| ---- | ---- | ---- | ---- |
| Authorization | string | 是 | 鉴权 access_token |
| Platform | string | 是 | 固定值：open_platform |

### （二）Body 参数
| 名称 | 类型 | 是否必填 | 说明 |
| ---- | ---- | ---- | ---- |
| fileIDs | array | 是 | 待删除文件的 ID 数组，一次性最多支持传入 100 个文件 ID |

## 三、请求示例（Curl）
```curl
curl --location 'https://open-api.123pan.com/api/v1/file/trash' \
--header 'Content-Type: application/json' \
--header 'Platform: open_platform' \
--header 'Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJl...(过长省略)' \
--data '{
"fileIDs": [
14705301,
14705306
]
}'
```

### 其他语言示例
支持 Java - OkHttp、JavaScript - jQuery、NodeJs - Axios、Python - http.client 等语言，具体实现可参考官方完整文档。

## 四、响应示例
```json
{
"code": 0,
"message": "ok",
"data": null,
"x-traceID": "566f23e5-9d27-400a-8e26-b3c8cc8ae6b0_kong-db-5898fdd8c6-wnv6h"
}
```

## 补充说明
- 响应中 `code: 0` 且 `message: "ok"` 表示文件删除请求执行成功，文件已移入回收站。
- 若传入的文件 ID 不存在或无操作权限，接口可能返回对应错误提示，具体错误信息可参考 `message` 字段。
- 单次请求传入的文件 ID 数量不能超过 100 个，超出限制会导致接口报错，需分批次处理大量文件删除操作。

# 彻底删除文件接口说明
## 一、接口基础信息
- **API 地址**：POST 域名 + /api/v1/file/delete
- **核心说明**：仅支持删除已在回收站中的文件，未移入回收站的文件无法通过该接口彻底删除，操作不可逆。

## 二、请求参数
### （一）Header 参数
| 名称 | 类型 | 是否必填 | 说明 |
| ---- | ---- | ---- | ---- |
| Authorization | string | 是 | 鉴权 access_token |
| Platform | string | 是 | 固定值：open_platform |

### （二）Body 参数
| 名称 | 类型 | 是否必填 | 说明 |
| ---- | ---- | ---- | ---- |
| fileIDs | array | 是 | 待彻底删除文件的 ID 数组，单次传入数量最大不超过 100 个 |

## 三、请求示例（Curl）
```curl
curl --location 'https://open-api.123pan.com/api/v1/file/delete' \
--header 'Content-Type: application/json' \
--header 'Platform: open_platform' \
--header 'Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJl...(过长省略)' \
--data '{
"fileIDs": [
14705301,
14705306
]
}'
```

### 其他语言示例
支持 Java - OkHttp、JavaScript - jQuery、NodeJs - Axios、Python - http.client 等语言，具体实现可参考官方完整文档。

## 四、响应示例
```json
{
"code": 0,
"message": "ok",
"data": null,
"x-traceID": "dbc1546b-b701-4d30-821a-c29283ffeac5_kong-db-5898fdd8c6-wnv6h"
}
```

## 补充说明
- 响应中 `code: 0` 且 `message: "ok"` 表示文件彻底删除操作执行成功。
- 前置条件校验：若传入的文件 ID 不在回收站中、文件不存在或无操作权限，接口会返回对应错误提示，具体可查看 `message` 字段。
- 数量限制：单次请求传入的文件 ID 数量不能超过 100 个，超出需分批次处理。
- 不可逆提示：该操作会永久删除文件，无法恢复，请谨慎使用。
