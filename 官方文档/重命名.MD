# 单个文件重命名接口说明
## 一、接口基础信息
- **API 地址**：PUT 域名 + /api/v1/file/name
- **核心用途**：用于修改 123 云盘中单个文件的名称

## 二、请求参数
### （一）Header 参数
| 名称 | 类型 | 是否必填 | 说明 |
| ---- | ---- | ---- | ---- |
| Authorization | string | 是 | 鉴权 access_token |
| Platform | string | 是 | 固定值：open_platform |

### （二）Body 参数
| 名称 | 类型 | 是否必填 | 说明 |
| ---- | ---- | ---- | ---- |
| fileId | number | 是 | 目标文件的唯一 ID |
| fileName | string | 是 | 新的文件名（需遵循云盘文件名命名规范，避免包含非法字符） |

## 三、请求示例（参考格式）
### Curl 示例（模板）
```curl
curl --request PUT \
--url "域名 + /api/v1/file/name" \
--header 'Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5c...(过长省略)' \
--header 'Platform: open_platform' \
--header 'Content-Type: application/json' \
--data '{
"fileId": 11522654,
"fileName": "新文件名.mp4"
}'
```

### 其他语言示例
支持 Java - OkHttp、JavaScript - jQuery、NodeJs - Axios、Python - http.client 等语言，具体实现可参考官方完整文档。

## 四、响应示例
官方未提供具体响应示例，通常接口请求成功会返回类似如下格式（供参考）：
```json
{
"code": 0,
"message": "ok",
"data": null,
"x-traceID": ""
}
```

## 补充说明
- 文件名需符合 123 云盘命名规则（通常要求：小于 256 个字符，不包含 \"\/:*?|>< 等非法字符，且不能全部为空格）。
- 若修改后的文件名与目标目录下现有文件重名，需确认是否有对应的重名处理策略（参考其他接口逻辑，可能默认不允许重名，具体以官方规则为准）。

# 批量文件重命名接口说明
## 一、接口基础信息
- **API 地址**：POST 域名 + /api/v1/file/rename
- **核心说明**：支持同时对多个文件进行重命名操作，最多可批量处理 30 个文件。

## 二、请求参数
### （一）Header 参数
| 名称 | 类型 | 是否必填 | 说明 |
| ---- | ---- | ---- | ---- |
| Authorization | string | 是 | 鉴权 access_token |
| Platform | string | 是 | 固定值：open_platform |

### （二）Body 参数
| 名称 | 类型 | 是否必填 | 说明 |
| ---- | ---- | ---- | ---- |
| renameList | array | 是 | 批量重命名列表，数组中每个成员格式为「文件ID|新的文件名」（需遵循云盘文件名命名规范） |

## 三、请求示例（参考格式）
### Curl 示例（模板）
```curl
curl --request POST \
--url "域名 + /api/v1/file/rename" \
--header 'Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5c...(过长省略)' \
--header 'Platform: open_platform' \
--header 'Content-Type: application/json' \
--data '{
"renameList": [
"11522654|新文件名1.mp4",
"11522655|新文件名2.txt"
]
}'
```

### 其他语言示例
支持 Java - OkHttp、JavaScript - jQuery、NodeJs - Axios、Python - http.client 等语言，具体实现可参考官方完整文档。

## 四、返回数据
### （一）整体结构
| 名称 | 类型 | 是否必填 | 说明 |
| ---- | ---- | ---- | ---- |
| successList | array | 是 | 重命名成功的文件列表 |
| failList | array | 是 | 重命名失败的文件列表（原文档此处描述有误，应为“失败文件列表”） |

### （二）successList 成员结构
| 名称 | 类型 | 是否必填 | 说明 |
| ---- | ---- | ---- | ---- |
| fileID | number | 是 | 重命名成功的文件唯一 ID |
| updateAt | string | 是 | 文件名称更新时间（格式通常为时间戳或标准时间字符串，以官方返回为准） |

### （三）failList 成员结构
| 名称 | 类型 | 是否必填 | 说明 |
| ---- | ---- | ---- | ---- |
| fileID | number | 是 | 重命名失败的文件唯一 ID |
| message | string | 是 | 重命名失败的具体原因（如文件名非法、文件不存在等） |

## 五、响应示例（参考格式）
```json
{
"code": 0,
"message": "ok",
"data": {
"successList": [
{
"fileID": 11522654,
"updateAt": "2024-11-07 09:10:30"
}
],
"failList": [
{
"fileID": 11522655,
"message": "文件名包含非法字符"
}
]
},
"x-traceID": ""
}
```

## 补充说明
- 数组中每个成员的「文件ID」与「新文件名」需用竖线（|）分隔，且新文件名需符合云盘命名规则（通常要求：小于 256 个字符，不包含 \"\/:*?|>< 等非法字符，且不能全部为空格）。
- 批量处理上限为 30 个文件，若超出限制，接口可能返回报错或仅处理前 30 个文件，需严格遵循数量限制。
- 若单个文件重命名失败（如文件名重复、文件不存在），不会影响其他文件的重命名操作，失败信息会集中在 failList 中返回。
